Git Permission Denied

使用git push命令推送文件时报错Permission Denied(publickey)
  1. Generate ssh key

首先，如果没有ssh key的话在控制台输入ssh-keygen -t rsa -C "xx@example.com"并执行。 途中会让输入密码不用管，直接一路回车直至生成你的ssh key。

注意：如果重新生成将会覆盖之前生成的ssh key

  1. 第二步执行命令ssh -v git@github.com，在最后面两句将出现：
  1. 第三步再执行ssh-agent -s会出现如下图所示信息。
  1. 接着输入ssh-add ~/.ssh/id_rsa后会提示

如果出现 Could not open a connection to your authentication agent. 的错误，先执行一下 eval `ssh-agent -s`,再继续执行ssh-add ~/.ssh/id_rsa基本上就可以了。

  1. 打开刚刚生成的id_rsa.pub，复制里面的内容，进入github账号的settings页面，在 SSH and GPG Keys 设置中找到 New SSH Key。

Title任意命个名，把刚才复制的内容粘贴进key中，完成后点击Add SSH Key。

  1. 最后使用ssh -T git@github.com验证Key，出现如下信息后就搞定了。
