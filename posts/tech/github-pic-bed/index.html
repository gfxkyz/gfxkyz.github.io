本文搭建图床的步骤主要分为如下三步。

Generate Token (classic)#

第一步，进入Github主页点击个人头像进入设置，根据下方地址找到生产Token的位置。（Developer Settings在设置列表的最后一个，注意往下滚动一下。）

Settings ➡️ Developer settings ➡️ Personal access tokens ➡️ Token (classic)

找到如上页面后，点击Generate New Token并输入Github账号的密码后进入创建Token页面。

Token过期时间按照默认的30天，过期了再次生成即可。（反正也不麻烦，还能防止Github密码忘记的惨剧发生。🤪）

Note是当前Token的注释，填写上与图片相关的名称即可，我在此处填写的图床的仓库名称。下方配置的勾选项只需勾选上如上图所示的Repo相关选项即可，勾选完毕后滑动到底部点击Generate Token按钮，这样第一步就完成了。

此处生成的Token会在PicGo配置时用到

Install PicGo on Mac#

我的系统环境是M2 MacOS Sequoia 15.0.1。

点击当前链接进入PicGo官网，或直接选择一个下方的下载源下载安装包。（表格截取自官网）

我选择的下载源是GitHub Release，当前最新正式版为2.3.1。滚动到下载位置下载下图高亮的安装包。

Arm芯片的Mac可能会出现无法打开或运行的问题（如上图），图中所示链接如下。

PicGo/FAQ.md at dev · Molunerfinn/PicGo

MacOS上安装完成运行后PicGo会在顶部Menu Bar显示图标，不会在Dock上显示。

点击Menu Bar上的图标打开主窗口进入到Github配置页面。

仓库名的格式为<github account>/<repo name>，

现在Github新建的仓库分支名称为main，不再是之前的master。

Token栏填上我们第一步获取到的Token。

自定义域名使用如下格式。JSDELIVR是一个适用于开源项目的免费 CDN，可以帮助我们加速访问图床。

https://cdn.jsdelivr.net/gh/账户名/仓库名@main

至此我们就可以通过PicGo上传图片等文件至远程仓库了。
